# Design: Settings Features

## Context

ユーザーが表示内容や投稿内容をカスタマイズできるようにするため、VSCodeの設定APIを使用して設定機能を実装します。

## Goals / Non-Goals

### Goals
- 表示項目のON/OFF切り替え
- 投稿内容のカスタマイズ（表示項目の選択）
- 設定の永続化
- 設定変更時のリアルタイム反映

### Non-Goals
- 複雑なカスタマイズ（色、フォントなど）
- 高度な設定UI（v1ではVSCode標準UIを使用）
- 設定のエクスポート/インポート（将来実装）

## Decisions

### Decision: VSCode設定APIを使用
- **理由**: 標準的な方法、永続化が自動、UIが提供される
- **代替案**: カスタム設定UI（複雑、実装コストが高い）

### Decision: 表示項目ごとにON/OFF設定
- **理由**: シンプルで理解しやすい、柔軟なカスタマイズ
- **代替案**: プリセット選択（柔軟性が低い）

### Decision: Slack投稿項目は複数選択可能
- **理由**: ユーザーが必要な情報のみを選択できる
- **実装方法**: 配列形式の設定項目

### Decision: 設定変更時のリアルタイム反映
- **理由**: ユーザーが即座に変更を確認できる
- **実装方法**: `workspace.onDidChangeConfiguration`イベント

## Risks / Trade-offs

- **リスク**: 設定の複雑化
- **対策**: シンプルな設定項目、明確な説明文

- **リスク**: 設定変更時のパフォーマンス
- **対策**: デバウンス、必要最小限の更新

## Migration Plan

新規機能のため、マイグレーションは不要です。既存の表示・投稿機能はデフォルト設定で動作します。
