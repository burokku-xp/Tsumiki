# Change: Fix Auto Session Creation

## Why

現在の実装では、タイマーを開始していない（アクティブなセッションが存在しない）場合、ファイル保存が記録されません。これにより、ユーザーがタイマーを開始するのを忘れた場合、作業内容が記録されず「本日のデータはありません」と表示されてしまいます。

また、自動作成されたセッションが長時間放置されると、作業時間の計算が不正確になる可能性があります。

ユーザー体験を向上させるため、以下の機能を追加します：
1. セッションが存在しない場合でも自動的にセッションを作成し、ファイル編集を記録できるようにする
2. 一定時間アクティビティがなかった場合、セッションを自動終了する
3. セッション自動作成時にユーザーに通知する

## What Changes

- ファイル保存時にセッションが存在しない場合、自動的にセッションを作成する機能を追加
- `WorkTimer`クラスに既存セッションを設定するメソッドを追加（外部からセッションを設定可能にする）
- セッションの自動作成時に`WorkTimer`の状態も更新する
- セッションが存在しない場合でもファイル編集が記録されることを保証
- 一定時間（デフォルト1時間）アクティビティがなかった場合、セッションを自動終了する機能を追加
- セッション自動作成時にユーザーに通知メッセージを表示する

## Impact

- Affected specs: `measurement` (MODIFIED)
- Affected code:
  - `src/extension.ts` (ファイル保存イベントハンドラーの修正、`getSession`と`createSession`のインポート追加、セッション自動作成通知、非アクティブセッション監視)
  - `src/measurement/timer.ts` (セッション設定メソッドの追加、非アクティブセッション監視機能)
