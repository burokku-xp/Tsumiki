# コードレビューチェックリスト

このチェックリストは、コードレビューを体系的に実施するための項目をまとめたものです。

## 機能性

### 要件の充足
- [ ] 要件を満たしているか
- [ ] 期待される動作を実現しているか
- [ ] エッジケースが適切に処理されているか
- [ ] 境界値が適切に処理されているか

### エラーハンドリング
- [ ] エラーケースが適切に処理されているか
- [ ] エラーメッセージが分かりやすいか
- [ ] エラー時のロールバックが適切か
- [ ] 例外処理が適切に実装されているか

### ユーザー体験
- [ ] ユーザーにとって使いやすいか
- [ ] 適切なフィードバックが提供されているか
- [ ] ローディング状態が適切に表示されているか
- [ ] エラー時のユーザーガイダンスが適切か

## セキュリティ

### 入力値の検証
- [ ] すべての入力値が検証されているか
- [ ] SQLインジェクション対策がされているか
- [ ] XSS対策がされているか
- [ ] CSRF対策がされているか

### 認証・認可
- [ ] 認証が適切に実装されているか
- [ ] 権限チェックが適切か
- [ ] セッション管理が適切か
- [ ] パスワードの取り扱いが適切か

### 機密情報
- [ ] 機密情報がコードにハードコードされていないか
- [ ] 環境変数が適切に使用されているか
- [ ] ログに機密情報が出力されていないか
- [ ] APIキーなどの認証情報が適切に管理されているか

## パフォーマンス

### データベース
- [ ] クエリが最適化されているか
- [ ] N+1問題がないか
- [ ] インデックスが適切に使用されているか
- [ ] 不要なデータ取得がないか

### メモリ管理
- [ ] メモリリークの可能性がないか
- [ ] リソースが適切に解放されているか
- [ ] 大きなデータ構造が適切に処理されているか

### 非同期処理
- [ ] 非同期処理が適切に実装されているか
- [ ] Promise/async-awaitが適切に使用されているか
- [ ] エラーハンドリングが適切か
- [ ] タイムアウト処理が適切か

### React固有
- [ ] 不要な再レンダリングがないか
- [ ] useMemo/useCallbackが適切に使用されているか
- [ ] コンポーネントの分割が適切か
- [ ] バンドルサイズが適切か

## 可読性

### 命名
- [ ] 変数名・関数名が適切か（意図が明確か）
- [ ] 命名規則に従っているか（camelCase、PascalCaseなど）
- [ ] 略語が適切に使用されているか
- [ ] マジックナンバーが定数化されているか

### コメント
- [ ] 複雑なロジックにコメントがあるか
- [ ] コメントがコードと一致しているか
- [ ] 日本語コメントが適切に記載されているか（プロジェクト規約）
- [ ] 不要なコメントが削除されているか

### コード構造
- [ ] コードの意図が明確か
- [ ] 関数が適切な長さか（単一責任の原則）
- [ ] ネストが深すぎないか
- [ ] コードの流れが理解しやすいか

## 保守性

### コードの重複
- [ ] コードの重複がないか（DRY原則）
- [ ] 共通化できる処理が共通化されているか
- [ ] ユーティリティ関数が適切に使用されているか

### モジュール化
- [ ] 適切にモジュール化されているか
- [ ] 依存関係が明確か
- [ ] 循環依存がないか
- [ ] 関心の分離が適切か

### 拡張性
- [ ] 将来の変更が容易か
- [ ] 設定が外部化されているか
- [ ] プラグイン可能な設計か
- [ ] 抽象化が適切か

### 型安全性（TypeScript）
- [ ] 型が適切に定義されているか
- [ ] any型が不必要に使用されていないか
- [ ] 型アサーションが適切か
- [ ] ジェネリクスが適切に使用されているか

## テスト

### テストの存在
- [ ] 単体テストが書かれているか
- [ ] 統合テストが適切か
- [ ] テストカバレッジが十分か

### テストの品質
- [ ] テストが保守しやすいか
- [ ] テストが明確か（何をテストしているか）
- [ ] テストが独立しているか
- [ ] モックが適切に使用されているか

### エッジケース
- [ ] エッジケースがテストされているか
- [ ] エラーケースがテストされているか
- [ ] 境界値がテストされているか

## プロジェクト固有

### OpenSpec
- [ ] 変更がOpenSpecのchange proposalに基づいているか
- [ ] specの要件を満たしているか
- [ ] 設計ドキュメントと一致しているか

### アーキテクチャパターン
- [ ] イベント駆動パターンが適切に使用されているか
- [ ] データ層の抽象化が適切か
- [ ] WebView通信が適切に実装されているか
- [ ] 機能分割が適切か

### コーディング規約
- [ ] TypeScript strict modeが有効か
- [ ] 命名規則に従っているか
- [ ] コメント規約に従っているか
- [ ] ファイル構造が適切か
